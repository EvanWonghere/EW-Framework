# EW Framework 文档决策记录

## 决策 001：采用“文档优先，不先编码”

- 日期：2026-03-14
- 背景：项目目标同时包含模板复用与 showcase 展示。
- 决策：先完成架构蓝图、模块规格、Demo 方案、README，再进入实现。
- 原因：避免模块语义漂移，减少返工，提高展示叙事一致性。

## 决策 002：保持 Hybrid 风格，避免过早绑定单一玩法

- 日期：2026-03-14
- 背景：用户选择 `Hybrid` 方向。
- 决策：文档中的模块定义保持中立，不绑定 Action RPG/MOBA/回合制特定机制。
- 原因：提高模板迁移性，便于后续按项目需求裁剪。

## 决策 003：采用“基建优先”推进顺序

- 日期：2026-03-14
- 背景：用户选择“基建优先”。
- 决策：先固化 Core/Modules 边界与规格，再设计 Demo 展示脚本。
- 原因：保证后续 Demo 不是一次性脚本，而是对框架能力的稳定映射。

## 决策 004：统一演示结构

- 日期：2026-03-14
- 决策：每个 Demo 都采用“目标 -> 步骤 -> 观察点 -> 成功判定”结构。
- 原因：确保演示可复现、可验收、可录制。

## 决策 005：README 对外定位

- 日期：2026-03-14
- 决策：README 明确声明项目当前处于文档规划阶段，并区分“当前能力”与“规划能力”。
- 原因：防止读者误判项目成熟度，提升对外沟通准确性。
